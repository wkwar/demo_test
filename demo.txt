这个仓库是用来 记录本地的一些修改

主要针对的是master分支，个人协作者的修改。这里没有合并test分支的数据。
原因：test分支用于测试 【包含了各种各样的代码，多个人协作】
===>其中可能有21期的需求变更，这个代码无法合并到只需要20期需求的master分支
==>所以需要一个单独的仓库来保证自己在20期需求的代码变更，后面合并方便。


多人协作Git代码开发
1.切自己分支my_demo，将变更提交到自己的远程分支【自己的存储样本】
2.切换 test分支，git pull origin test 获取最新的test环境仓库代码 【拉取test环境最新代码】
3.将test分支与本地分支合并，get merge my_demo 【与公司test环境代码合并】
3.将合并的代码推送到test分支 git push origin test 【变更提交到test环境】

==>[
    1.保证自己的代码只有自己的代码变更，没有其他人协作上的变更 [保证自己的变更清楚，更方便合并到master分支]
    2.完成test分支代码 与 自己代码变更的合并 【完成测试环境的多人代码协作】
]
